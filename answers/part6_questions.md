# Часть 6: Контрольные вопросы

## 1) В каком порядке выполняются конструкторы при создании объекта подкласса?
_Ваш ответ:_
Сначала вызывается конструктор суперкласса (родителя), затем — конструктор подкласса.

    class Animal {
        Animal() { System.out.println("Конструктор Animal"); }
    }

    class Dog extends Animal {
        Dog() { System.out.println("Конструктор Dog"); }
    }

    new Dog(); 

Ошибка новичка: Забывают вызвать super() явно, но Java вызывает его автоматически (если нет явного вызова).

## 2) Можно ли переопределить `static` метод? Что такое method hiding?
_Ваш ответ:_
Нельзя переопределить static-метод. Если в подклассе написать метод с такой же сигнатурой — это сокрытие метода (method hiding). Какой метод вызовется, зависит от типа переменной во время компиляции.

    class Parent {
        static void print() { System.out.println("Parent"); }

    }
    class Child extends Parent {
        static void print() { System.out.println("Child"); }
    }

    Parent p = new Child();
    p.print(); 

Ошибка: Думают, что static работает как обычное переопределение.

## 3) Что произойдет, если в `switch` нет `default` и ни один `case` не совпал?
_Ваш ответ:_
Ничего не произойдёт — код внутри switch просто пропустится, программа продолжит выполнение дальше.

    int x = 10;
    switch (x) {
        case 1: System.out.println("1"); break;
        case 2: System.out.println("2"); break;
    }

Ошибка: Новички думают, что будет ошибка компиляции.


## 4) Почему в `for-each` нельзя удалять элементы коллекции? Как правильно?
_Ваш ответ:_
for-each использует внутренний итератор. При удалении элемента структура меняется, и итератор "ломается" — выбрасывается ConcurrentModificationException.

Как правильно: Использовать Iterator с методом remove() или собственный цикл.

    Iterator<String> it = list.iterator();

    while (it.hasNext()) {

            if (it.next().equals("удалить")) {

                it.remove(); 

            }

    }

## 5) Чем отличается `break` с меткой от обычного `break`?
_Ваш ответ:_
Обычный break выходит только из текущего (внутреннего) цикла. break с меткой выходит из помеченного внешнего цикла сразу.

    outer:
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            if (i == 1 && j == 1) break outer; // Выход из обоих циклов
        }
    }

Ошибка: Пытаются выйти из внешнего цикла без метки.

## 6) Что означает `@Override`?
_Ваш ответ:_
Аннотация, которая показывает, что метод переопределяет метод суперкласса или реализует метод интерфейса. Помогает компилятору найти ошибку (если неправильно написали имя метода).

    @Override
    public String toString() { ... } // Переопределение

Ошибка: Без @Override можно случайно написать toStrring() — будет новый метод, а не переопределение.

## 7) Какой модификатор доступа выбрать для видимости в подклассах?
_Ваш ответ:_
protected — поля/методы видны в подклассах (даже в другом пакете) и в своём пакете.


    Модификатор	В классе	В подклассе	В пакете	Вне пакета
    private	        ✅	            ❌	            ❌	            ❌
    protected       ✅	            ✅	            ✅	            ❌
    public	        ✅	            ✅	            ✅	            ✅

Ошибка: Используют private и потом удивляются, что в подклассе не видно.

## 8) Разница между `final` классом и `sealed` классом?
_Ваш ответ:_
final класс — нельзя наследовать (создать подкласс) вообще.

sealed класс — разрешает наследование только указанным классам.

    sealed class Animal permits Dog, Cat { }
    final class Dog extends Animal { }    
    final class Cat extends Animal { }     

Ошибка: Думают, что sealed делает то же, что final.

## 9) Можно ли иметь несколько конструкторов?
_Ваш ответ:_
Да, это называется перегрузка конструкторов. Конструкторы отличаются количеством или типами параметров.

    class Person {
        String name;
        int age;
        Person(String name) { this.name = name; age = 0; }
        Person(String name, int age) { this.name = name; this.age = age; }
    }

Ошибка: Пытаются создать конструктор с одинаковыми параметрами.


## 10) Что такое ковариантный тип возвращаемого значения?
_Ваш ответ:_
Это возможность при переопределении метода возвращать более конкретный (подкласс) тип, чем в суперклассе.

    class Animal { Animal get() { return this; } }
    class Dog extends Animal {
        @Override
        Dog get() { return this; } 
    }
---
Ошибка: Думают, что тип должен быть точно таким же.
